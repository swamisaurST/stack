# Workflow: Audit

Evaluate an existing Sitetracker design, prototype, implementation, screenshot, Storybook story, LWC, Pencil file, or Figma frame. Review only — no new design output unless the designer switches scope.

## Task Progress

- [ ] STACK activated
- [ ] Audit target and platform confirmed
- [ ] If auditing an existing project folder, write findings inside it; create no new folder
- [ ] Platform file + refs for each audited pattern loaded
- [ ] Findings listed by severity
- [ ] `HANDOFF.md` filled (lead with findings)

## Audit Focus

Scaffold integration and output location; header shell and title-row alignment; layout model; section/card header anatomy; icon source, size, meaning; table/list density; component reuse; state coverage; accessibility; visual-verification quality.

## Rules

- Lead with findings, ranked by severity.
- Compare control-level against `references/GALLERY-REFERENCE.md` for shell/icon claims (Law 9).
- Do not produce new design unless the designer explicitly switches from audit to build.
