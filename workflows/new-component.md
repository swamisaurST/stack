# Workflow: New Component

Use only when STACK references and SLDS 2 do not cover the needed pattern and the designer approves exploring a new component.

## Task Progress

- [ ] STACK activated
- [ ] Platform, destination, output location confirmed
- [ ] Verified STACK + SLDS 2 insufficient (`references/web/STACK-WEB-COMPONENT-PRECEDENCE.md`)
- [ ] End user, job, inputs, outputs, ownership confirmed
- [ ] Anatomy, variants, states, accessibility, responsive behavior, verification defined
- [ ] Designer approved direction before implementation
- [ ] `HANDOFF.md` filled

## Required Definition

Purpose; when to use; when not to use; anatomy; variants; required states; icon rules; accessibility; implementation target; visual-verification method; known gaps.

## Stop Conditions

Stop when an existing STACK or SLDS 2 pattern fits, the designer has not approved, states or accessibility are unclear, or the implementation target is unconfirmed.
