# Workflow: Transition

Move an existing design or implementation into the current STACK design language.

## Task Progress

- [ ] STACK activated
- [ ] Source, platform, destination, output location confirmed
- [ ] Project folder confirmed; path in `CONTEXT.md`
- [ ] Source audited against STACK references before transitioning
- [ ] What stays / changes / needs clarification identified
- [ ] Target layout model and component refs selected
- [ ] Designer approved major structural changes
- [ ] `HANDOFF.md` filled

## Transition Rules

- Preserve user workflow intent unless the designer approves a change.
- Replace unsupported visual patterns with STACK references.
- Do not carry over arbitrary centered shells, oversized row icons, ad hoc headers, or unverified icons.
- Treat missing source details as gaps, not creative freedom (Law 8).
