# Workflow: Prototype

Build a multi-screen local React review prototype, hash-routed, under `design/prototypes/<project-name>/`.

## Task Progress

- [ ] STACK activated
- [ ] Project folder `design/prototypes/<project-name>/` confirmed; path in `CONTEXT.md`
- [ ] Platform (web) and review format confirmed
- [ ] Shell, layout, and nav rail loaded (`references/web/STACK-WEB-GLOBAL-HEADER-COMPACT.md`, `references/web/STACK-WEB-COMPONENT-TITLE-GROUP.md`, `references/web/STACK-WEB-LAYOUT.md`, `references/web/STACK-WEB-PROTOTYPE-NAV-RAIL.md`)
- [ ] Gallery + visual protocol loaded if shell is included (`references/GALLERY-REFERENCE.md`, `references/VISUAL-VERIFICATION.md`)
- [ ] Hierarchy written per screen
- [ ] Prototype runs; page + control-level screenshots captured
- [ ] `HANDOFF.md` filled with part-screenshot evidence

## Rules

- Reuse the locked two-part shell; do not redraw it.
- Prototype lives outside `design/stack/` (Law 10).
- No `Result: Pass` without control-level visual evidence (Law 9).
